# Intelligent Medical Diagnosis Algorithm API Documentation

## 1. Overview

This interface provides multi-modal medical auxiliary diagnosis services based on a 28-specialty Expert Pool and Large Language Models (LLM). It accepts patient medical records (text or structured data), bowel sound prediction results, and electrocardiogram (ECG) prediction results. Through department routing, expert diagnosis, and multi-expert consultation, it outputs diagnostic suggestions for patients and professional risk assessments for doctors.

**Core Functions:**
*   **Multi-modal Fusion**: Integrates text records, ECG, and bowel sound data.
*   **Intelligent Triage**: Automatically matches the most suitable experts from 28 specialties.
*   **Dual-View Output**: Provides patient-friendly suggestions and professional management data for doctors.
*   **Risk Stratification**: Five-level risk assessment based on emergency triage standards.

## 2. Command Line Usage

Use the Python script `src/diagnosis_api.py` (relative to `evomed` root).

**Basic Command:**

```bash
python3 src/diagnosis_api.py [arguments...]
```

## 3. Arguments

| Argument | Type | Required | Default | Meaning |
| :--- | :--- | :--- | :--- | :--- |
| `--patient_file` | str | **Yes** | - | **Patient Record File Path**.<br>JSON format containing patient info, chief complaint, present illness, history, and lab results.<br>Supports multiple structures (see below). |
| `--c_file` | str | No | None | **Bowel Sound Prediction File Path**.<br>JSON format containing bowel sound model predictions.<br>If omitted, bowel sound factors are ignored. |
| `--ecg-file` | str | No | None | **ECG Prediction File Path**.<br>JSON format containing ECG model predictions.<br>If omitted, ECG factors are ignored. |

## 4. Input File Formats

### 4.1 Patient Record File (`--patient_file`)

Supports two JSON structures: **Standard Flat Format** (Recommended) and **Table Nested Format**.

**Example A: Standard Flat Format (GPT Output Format)**

Suitable for structured records generated by LLM.

```json
{
  "patientName": "Huiyuan Wen",       
  "patientGender": "Male",         // or "M", "男"
  "patientAge": 67,             // or "age": 67
  "chiefComplaint": "Sudden abdominal pain for 5 hours", 
  "presentIllness": "Patient is a 67-year-old male... (details)", 
  "personalHistory": "Denies digestive system diseases...",        
  "labs": "WBC 11.2, ALT 45...", // String or List
  "clinicCode": "pre-jqxci99"    // (Optional) Clinic/Session code for doctor output
}
```

**Example B: Table Nested Format**

Suitable for data submitted directly from frontend forms.

```json
{
  "gender": "M",                 
  "age": 67,                     
  "history": {                   
    "Chief Complaint": "Abdominal pain...", 
    "History of Present Illness": "..."
  },
  "labs": [                      
    {
      "key": "Blood Routine",            
      "value": "WBC: 11.2, RBC: 4.5..." 
    }
  ],
  "exam": [                      
    {
      "key": "Abdominal X-ray",
      "value": "No free gas under diaphragm..."
    }
  ],
  "physical_examination": {      
    "T": "37.2℃",
    "Abdomen": "Tenderness in right upper quadrant(+)"
  }
}
```

### 4.2 Bowel Sound Result File (`--c_file`)

JSON file generated by audio analysis model.

```json
{
  "fold": "fold_0",
  "pid": "TEST_001",
  "pred": 0,           // Prediction: 0=Normal, 1=Abnormal
  "prob_0": 0.72,      // Probability of Normal (0-1)
  "prob_1": 0.28       // Probability of Abnormal (0-1)
}
```

### 4.3 ECG Result File (`--ecg-file`)

JSON file generated by ECG analysis model. 
**Note:** Format aligned with Bowel Sound JSON.

```json
{
  "fold": "fold_0",    // (Optional)
  "pid": "TEST_001",
  "pred": 0,           // Prediction: 0=Normal, 1=Abnormal
  "prob_0": 0.85,      // Probability of Normal (0-1)
  "prob_1": 0.15       // Probability of Abnormal (0-1)
}
```

## 5. Output Files

The algorithm generates two JSON files in the **current execution directory**.

### 5.1 Patient Result (`diagnosis.json`)

Contains patient-friendly diagnostic suggestions, basis, and full risk stratification.

**Usage**: Frontend display for patients.

**Structure Example:**

```json
{
  "status": "success",
  "patient_info": {
    "diagnosis_top5": "1. Acute Appendicitis\n2. Mesenteric Lymphadenitis...", 
    "diagnosis_basis": "Right lower quadrant tenderness...",           
    "differential_diagnosis": "Need to exclude ureteral stones...",          
    "suggested_examinations": "Suggest abdominal ultrasound..."               
  },
  "risk_assessment": {
    "risk_level": "Emergency (Level III) - Consult within 30 mins",              
    "home_monitoring": "Observe temperature changes...",                 
    "visit_advice": "Suggest immediate visit to ER...",                  
    "emergency_needed": "No",                               
    "response_procedure": "..."                             
  },
  "expert_opinions": [...],  // (List) Detailed opinions from consulting experts
  "raw_output": "..."        // (String) Raw model output text
}
```
